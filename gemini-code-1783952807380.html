Spring Boot Project Structure

In this module, we dissect the absolute file framework generated by the Spring Initializr tool, focusing on the roles of Maven Wrappers, application properties, and Spring Boot Starter dependencies.

Spring Boot adopts the standardized Maven directory hierarchy conventions right out of the box. This provides consistency across your architectural components, making it straightforward to manage source configurations and executable scripts.

The Complete Directory Layout

Below is the detailed visual layout of a fresh Spring Boot project blueprint generated by the initialization system:

  • my-app/
    • src/
      • main/
        • java/ ← Core Java app source files
        • resources/
          • application.properties ← Target environment configurations
      • test/
        • java/ ← Unit testing code files (JUnit, Mockito)
    • mvnw ← Maven execution wrapper shell script (Linux / macOS)
    • mvnw.cmd ← Maven execution wrapper batch script (Windows)
    • pom.xml ← Master Maven manifest artifact
Demystifying the Maven Wrapper (mvnw)

The files mvnw and mvnw.cmd represent the **Maven Wrapper**. This wrapper lets you build and run your project securely without needing Maven manually installed or configured on your system's global system path variable.

The wrapper automatically scans your workspace. If it doesn't find the correct Maven runtime, it downloads the necessary version dynamically from remote mirrors before executing your instructions. If you already have the correct version of Maven globally installed on your machine, you can safely ignore or delete these wrapper files.

The Power of Spring Boot "Starters"

Instead of forcing you to list dozens of individual library declarations one by one inside your pom.xml file, Spring Boot introduces **Starters**. These are curated collections of aggregated, production-tested dependencies.

Starter Spotlight: spring-boot-starter-web

By declaring just this single starter artifact, Spring Boot automatically imports and configures several transitive libraries behind the scenes, including: Spring Web MVC, Jackson JSON Processing, Hibernate Validator, and an embedded Apache Tomcat Server.

This ensures version compatibility across your external tools and saves you hours of tracking down manual conflicts.

Command Line Build Operations

The spring-boot-maven-plugin handles compiling, packaging, and running your final application package directly from your integrated system terminal console:

Option A: Using the Maven Wrapper (No local Maven installation required)

# Windows Command Prompt
mvnw.cmd package
mvnw.cmd spring-boot:run

# Linux / macOS Terminal
./mvnw package
./mvnw spring-boot:run

Option B: Using Native Maven (If Maven is installed locally on your system path)

mvn package
mvn spring-boot:run
