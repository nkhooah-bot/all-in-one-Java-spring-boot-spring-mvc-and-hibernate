Spring Initializr & Maven Introduction

In this lecture, we will demonstrate how to bootstrap a clean Spring Boot application framework using Spring Initializr and seamlessly import it into your development ecosystem.

What is Maven?

When creating complex Java projects, manually downloading external .jar library dependencies (like Spring Web, Hibernate, or JSON parsing utilities) and configuring classpath properties can be highly error-prone. Maven serves as your automated virtual project assistant: you simply provide a list of declarations in a pom.xml file, and Maven fetches, compiles, and dynamically binds those resources at runtime.

Step 1: Generate the Project Structure
  • 1 Navigate to the browser web portal at start.spring.io.
  • 2 Select Maven under the build automation configuration profile options and assign Java as the application language.
  • 3 Choose the most recent stable release version of **Spring Boot**. (Avoid choosing variants labeled with "SNAPSHOT", as they represent experimental pre-release builds).
  • 4 Configure the Project Metadata variables exactly as follows:
    • Group ID: com.example.springboot.demo
    • Artifact ID: my-app (This represents the base project directory folder name)
  • 5 Click the **Add Dependencies** button on the right, search for Web, and choose **Spring Web**. This pulls in full-stack components, including Spring MVC and an embedded Tomcat server instance.
  • 6 Click the **Generate** button at the base of the page layout to compile your custom starter package file download (my-app.zip).
Step 2: Extract and Localize the Project Files
  • 1 Open your operating system's local file manager directory and track down the downloaded zip compression file.
  • 2 Unzip/uncompress the file pack contents completely.
  • 3 Copy the extracted project root folder and paste it into your desired active local workspace directory folder.
Step 3: Import into IntelliJ IDEA
  • 1 Launch your IDE (IntelliJ IDEA) and choose the **Open** action link prompt from the navigation dashboard window.
  • 2 Browse over and highlight your localized my-app root workspace folder, then click **OK**.
  • 3 When prompted by the system modal utility notification window, click **Trust Project**.
  • 4 Allow the framework internal automation loops time to run silently. Maven will scan your project configuration parameters and download all necessary remote dependency file wrappers.
Step 4: Analyze and Launch the Main Application Class

Inside the IDE, browse down the workspace directory structure tree to locate your main bootstrap class entry file: src/main/java/com/example/springboot/demo/MyappApplication.java.

@SpringBootApplication public class MyappApplication { public static void main(String[] args) { SpringApplication.run(MyappApplication.class, args); } }
Code Breakdown

@SpringBootApplication flags this target Java class profile blueprint structure as the official application entry gate template indicator hook.

SpringApplication.run() invokes the backend runtime orchestration module layers, launching your internal web server container automatically.

  • 1 To run the engine, click on the **Green Run Arrow Button** placed directly inside the main script text wrapper, or right-click your class source file body structure canvas and select **Run 'MyappApplication'**.
  • 2 Troubleshooting Tip: If the IDE interface elements fail to display context actions due to initialization indexing issues, right-click the root src folder workspace item directory inside your project view panel and select **Mark Directory as -> Sources Root** to force re-indexing.
