Lab: Actuator Security Enforcement

Protecting production diagnostic endpoints by integrating the Spring Security starter.

1. Dependency Injection

Add the security starter to your pom.xml to enable the automatic credential challenge filter on all Actuator endpoints.

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>
2. Handling Credentials

Upon application startup, examine the terminal console. Spring Boot prints a temporary, randomly generated password that serves as your initial authentication token.

3. Advanced Endpoint Exclusion

If specific diagnostic endpoints (e.g., /health or /info) should be blocked entirely, use the exclude property in application.properties:

# Blacklisting sensitive endpoints from external access
management.endpoints.web.exposure.exclude=health,info
4. Lab Verification
  • 1 Run the App: Start the application and copy the generated password from the console log.
  • 2 Test Access: Navigate to /actuator/mappings in your browser. Note the challenge prompt.
  • 3 Authenticate: Use username user and the password from your console logs to gain access.
