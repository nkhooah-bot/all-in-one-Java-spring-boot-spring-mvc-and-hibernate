Verifying the Server & Embedded Tomcat

Now that our application is initialized and running inside our IDE, let's explore how to inspect execution logs, verify server ports, and interpret early web browser output.

Once you run the application main method, Spring Boot instantiates the background framework ecosystem. Unlike traditional Java web developments, you don't need to manually configure separate server installations.

Analyzing the Startup Logs

Look at the console log outputs inside your IDE window. You will notice explicit initialization entries that verify your runtime context:

[main] o.s.b.w.embedded.tomcat.TomcatWebServer : Tomcat initialized with port(s): 8080 (http) [main] o.s.b.w.className.MyappApplication : Started MyappApplication in 2.134 seconds...
What is an Embedded Server?

Spring Boot includes an Embedded Tomcat Server package bundled directly within its starter frameworks. This means your application acts as an independent, self-contained executable that handles web routing endpoints without needing any external server installations.

Accessing Localhost
  • 1 Ensure your IDE console shows that the application is actively running.
  • 2 Open any modern web browser interface.
  • 3 In the URL navigation box, input the following target path address: http://localhost:8080
Expected Behavior: Whitelabel Error Page

When you visit the local server page right now, your browser will display a standard "Whitelabel Error Page" (HTTP Status 404 – Not Found). Do not panic! This indicates your server is successfully up and listening for incoming connection requests. The error displays purely because we haven't mapped any logic pathways or written codes to render data to this route yet.

Basic verification success! In our next module, we'll implement a custom REST Controller to override this error template and present data to the user.
