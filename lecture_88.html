Spring Dependency Injection (DI)

Implementing the Dependency Inversion principle to build modular applications.

What is Dependency Injection?

Dependency Injection (DI) is the practice of outsourcing object construction. Instead of your classes manually instantiating their dependencies (like "parts of a laptop"), you ask the Spring Container to provide them. The container assembles all necessary components and delivers the ready-to-use object to your application.

Two Primary Injection Types
  • Constructor Injection: Used for required dependencies. This is the approach officially recommended by the Spring team.
  • Setter Injection: Used for optional dependencies, allowing your application to fall back on default logic if a dependency is not provided.
Spring Auto Wiring

Spring simplifies DI through Auto Wiring. By scanning your application for components (classes annotated with @Component), Spring automatically identifies classes that match the required dependency type (e.g., matching an interface to its concrete implementation) and wires them together.

Key Takeaway

The Spring Container performs two core duties: Inversion of Control (creating/managing beans) and Dependency Injection (injecting those beans into other objects).
