Application Properties & Static Assets

In this lecture, we explore environment configuration tuning via application.properties, property value injection using @Value, and alternative asset routing patterns.

By default, a Spring Boot application automatically evaluates externalized initialization properties out of a dedicated environment entry file called application.properties, located inside your src/main/resources/ folder directory.

1. Setting Properties & Value Injection

While generated completely empty by Spring Initializr, you can populate this manifest with built-in framework keys (such as changing the default server listening port) or append custom property key-value pairs:

# Customizing the server port environment variable server.port=7070 # Defining custom corporate property boundaries instructor.name=John Doe class.name=Java Masterclass

To pull these custom values out of your properties payload and assign them directly into a REST Controller instance, use the spring-managed @Value annotation combined with standard ${property.key} expression wrappers:

@RestController public class DemoController { @Value("${instructor.name}") private String instructorName; @Value("${class.name}") private String className; @GetMapping("/info") public String getDetails() { return "Course: " + className + " | Instructor: " + instructorName; } }
2. Handling Static Content & The Webapp Pitfall

For rendering assets such as static images, custom CSS sheets, web scripts, or PDFs, Spring Boot automatically configures an internal static resource mapping directory pointing directly to src/main/resources/static/.

Critical Warning: Avoid src/main/webapp

Although src/main/webapp is a legacy standard directory structure folder for general Maven web apps, it only resolves properly when your project uses .war packaging deployment builds. If your app targets a standard executable embedded standalone .jar, build engines skip this path entirely. Utilizing it will lead to broken links and hours of debugging.

3. Baseline Testing Frameworks

The initialization engine also provisions a basic smoke test stub under src/test/java/. It contains a standard boilerplate layout designed to verify that the target Spring ApplicationContext loads without structural errors:

@SpringBootTest class MyApplicationTests { @Test void contextLoads() { // Verifies application boot integrity out of the box } }

While this template checks context booting integrity, you should expand this test class with custom unit tests as your operational features grow.
