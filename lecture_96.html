Setter Injection in Spring Boot

Mastering dependency injection via setter methods for modular Spring applications.

Implementation Steps
  • 1Create Setter Method: Define a public method to accept the dependency.
  • 2Add @Autowired: Annotate the method to allow Spring to automatically inject the required bean.
Code Example
@RestController
public class CourseController {
    
    private Instructor myInstructor;

    // Any method name works as long as it is @Autowired
    @Autowired
    public void setInstructor(Instructor instructor) {
        this.myInstructor = instructor;
    }

    @GetMapping("/programmingexercise")
    public String getProgrammingExercise() {
        return myInstructor.getProgrammingExercises();
    }
}
Testing Your Setup

1. Startup: Run your application. Spring will instantiate the controller and call the setter method automatically.

2. Validate: Navigate to http://localhost:8080/programmingexercise to confirm the dependency is active.

3. Hot-Reload: Use Ctrl+S (with DevTools) to modify and verify real-time injection updates without full restarts.

When to use Setter Injection?

While constructor injection is the industry standard for required dependencies, setter injection is the preferred choice for optional dependencies where you need the flexibility to modify or omit the injected component.
