Custom Application Properties

Externalizing configuration allows you to define flexible application settings inside application.properties and inject them seamlessly using the @Value annotation.

Spring Boot simplifies environment management by reading configuration from a standard file located in src/main/resources/application.properties. By defining custom keys here, you avoid hard-coding values into your Java source files.

1. Defining Custom Properties

Open your application.properties file and register your custom configuration keys:

# Custom application settings instructor.name=John Smith student.class=Java Class J02
2. Injecting Properties with @Value

Use the @Value annotation to map these properties to fields within your REST controllers. Spring Boot automatically resolves these placeholders during bean initialization:

@RestController public class SimpleRestController { // Injecting instructor name from application.properties @Value("${instructor.name}") private String instructorName; // Injecting student class name @Value("${student.class}") private String studentClass; @GetMapping("/teaminfo") public String getTeamInfo() { return "Instructor: " + instructorName + ", Class: " + studentClass; } }
Why use this pattern?

This approach decouples your business logic from environment-specific configuration. It makes your code cleaner and significantly easier to port between development, staging, and production environments.
