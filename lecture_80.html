Coding Lab: Customizing the Actuator Info Endpoint

Follow this hands-on setup guide to append custom metadata keys inside application.properties and verify the structured JSON telemetry details in real time.

While the /actuator/info path is exposed, calling it right now returns an empty JSON object ({}). In this coding session, we will configure custom corporate descriptor tags using the info.* namespace to supply metadata for the telemetry interface.

Step-by-Step Lab Execution Pipeline
  • 1 Update your properties block: Navigate to your src/main/resources/application.properties file and append your custom application info variables directly beneath your existing endpoint exposures:
# Expose select web endpoints (health and info) management.endpoints.web.exposure.include=health,info management.info.env.enabled=true # Custom Metadata properties for /actuator/info endpoint info.app.name=Healthy Activities info.app.description=This app keeps track of your daily healthy activities. info.app.version=3.0.0

Tip: Because you configured Spring Boot DevTools in the earlier modules, saving this file triggers an instant background context refresh—no manual restarts required!

  • 2 Save and Monitor the Console: Once you save the configurations, check your IDE's active run terminal window. You'll observe the classpath changes being loaded seamlessly in real-time.
Verifying the Structured JSON Payload

Return to your web browser and execute a clean refresh on your endpoint. The empty array is replaced with nested, well-formed configuration objects:

http://localhost:8080/actuator/info
{
  "app": {
    "name": "Healthy Activities",
    "description": "This app keeps track of your daily healthy activities.",
    "version": "3.0.0"
  }
}
Under the Hood: The info.* Namespace Mapping

When management.info.env.enabled is flagged as true, the Actuator engine scans your environmental scope, isolates all values prefix-matched with info., structures the keys recursively, and formats them into the JSON tree response context automatically.
