Spring Boot Component Scanning

Understanding how Spring discovers and registers your beans.

The Default Scanning Scope

Spring Boot automatically scans components (classes with @Component, @RestController, etc.) starting from the package containing your main @SpringBootApplication class. It recursively scans all subpackages below that root.

Explicit Package Scanning

If your components are located in packages outside the root package hierarchy, Spring will not find them by default. You must explicitly configure the scanBasePackages attribute:

@SpringBootApplication(scanBasePackages = {
    "com.example.springcoretutorial",
    "com.example.springbootsecurity",
    "org.domainname.example"
})
public class SpringCoreTutorialApplication {
    public static void main(String[] args) {
        SpringApplication.run(SpringCoreTutorialApplication.class, args);
    }
}
Key Annotations

The @SpringBootApplication annotation is a convenience wrapper for:

  • @EnableAutoConfiguration: Enables auto-configuration support.
  • @ComponentScan: Enables component scanning.
  • @Configuration: Allows for manual bean registration via @Bean methods.
Keep your packages structured under the root directory whenever possible to leverage default scanning.
