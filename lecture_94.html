Spring Boot Component Scanning

Mastering automated bean discovery and custom package registration.

Spring Boot simplifies application setup by automatically scanning your Java classes for annotations like @Component, @RestController, and @Service to register them as Beans in the Spring Container.

The "Magic" Annotation: @SpringBootApplication

This is a convenience annotation that combines three critical features:

  • @EnableAutoConfiguration: Activates Spring Boot’s intelligent auto-config.
  • @ComponentScan: Tells Spring to look for components in the current package and sub-packages.
  • @Configuration: Tags the class as a source of bean definitions.
Scanning Strategy Comparison

Default Scanning

  • Implicitly starts from the package of the main class.
  • Automatically scans all sub-packages underneath.
  • Requires no additional code.

Explicit Scanning

  • Used when components live in external packages.
  • Requires the scanBasePackages attribute.
  • Overrides the default single-package limit.
Implementation: Explicit Scanning

When your structure includes packages like com.example.edu that are NOT sub-packages of your main app, use the following configuration:

package com.example.springcoretutorial;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication(scanBasePackages = {
    "com.example.springcoretutorial", 
    "com.example.edu",
    "org.university.utils"
})
public class SpringCoreTutorialApplication {

    public static void main(String[] args) {
        SpringApplication.run(SpringCoreTutorialApplication.class, args);
    }
}
Troubleshooting & Logs

If Spring cannot find your bean (resulting in a BeanDefinitionStoreException), verify these common points:

  • Is the class annotated with @Component or its stereotypes?
  • Is the package listed correctly in the scanBasePackages array?
  • Check the Tomcat Console: Successful scanning will show mapped endpoints in the logs during startup.
Best Practice: Keep your project hierarchical under one root package to take full advantage of Spring's default "Zero-Config" scanning.
