Constructor Injection in Spring Boot

A comprehensive guide to implementing decoupled components using Spring's dependency injection.

Workflow Overview
  • 1Define Interface: Create an Instructor interface.
  • 2Implement Bean: Create JavaInstructor annotated with @Component.
  • 3Build Controller: Inject the dependency via CourseController constructor using @Autowired.
  • 4Expose Endpoint: Map the functionality with @GetMapping.
Implementation Code

Step 1: Interface Definition

public interface Instructor {
    String getProgrammingExercises();
}

Step 2: Bean Implementation

@Component
public class JavaInstructor implements Instructor {
    @Override
    public String getProgrammingExercises() {
        return "Write a method that sums two numbers.";
    }
}

Step 3 & 4: Controller Logic

@RestController
public class CourseController {
    private Instructor myInstructor;

    @Autowired
    public CourseController(Instructor instructor) {
        myInstructor = instructor;
    }

    @GetMapping("/programmingexercise")
    public String getProgrammingExercise() {
        return myInstructor.getProgrammingExercises();
    }
}
How to Test Your Code

1. Run: Start your application from your IDE. Check the log for a successful Tomcat startup on port 8080.

2. Access: Open http://localhost:8080/programmingexercise in your browser to see the output.

3. Live Reload: Change the return string in JavaInstructor, save the file (Ctrl+S), and refresh the browser to see updates instantly, thanks to spring-boot-devtools.

Note: If hot-reload fails, verify "Build project automatically" is enabled in your IDE's Advanced Settings.

Success! Your modular, decoupled Spring Boot application is now fully operational.
