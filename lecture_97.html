Field Injection in Spring Boot

Understanding legacy dependency injection and why it is no longer recommended.

What is Field Injection?

Field injection uses the @Autowired annotation directly on a class field. Spring uses Java Reflection to set the field value, even if it is private, completely bypassing constructors or setter methods.

Code Implementation
@RestController
public class CourseController {

    // Dependency is injected directly via reflection
    @Autowired
    private Instructor myInstructor;

    @GetMapping("/programmingexercise")
    public String getProgrammingExercise() {
        return myInstructor.getProgrammingExercises();
    }
}
Why it is Not Recommended
  • Hard to Test: You cannot easily mock dependencies because the fields are private and inaccessible without complex reflection.
  • Hidden Dependencies: Requirements for the class are not visible in the constructor.
  • Tight Coupling: The class becomes heavily dependent on the Spring framework for initialization.
Recommended Alternatives

To follow modern best practices, prioritize these injection types:

  • Constructor Injection (Recommended): Best for required dependencies. Enables immutability.
  • Setter Injection: Best for optional dependencies. Provides flexibility.
Legacy Warning: You may find Field Injection in older codebases, but modern Spring development favors explicit constructor-based dependency injection.
