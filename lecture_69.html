Creating Your First REST Controller

In this module, we will replace the default Spring Boot Whitelabel error page by implementing a simple REST controller endpoint that returns a custom string response directly to the browser view.

Currently, running the server shows an error page at localhost:8080 because no custom route mapping has been explicitly built. Before adding new code files, ensure your server is stopped by clicking the Red Square Button in your IDE console.

Step 1: Create a New Sub-package
  • 1 Inside your project file structure navigation tree, locate your root code package location: src/main/java/com/example/springboot/demo.
  • 2 Right-click on the package folder, select New -> Package.
  • 3 Name your child directory suffix package target extension: rest. (The absolute structural context pathway expands to com.example.springboot.demo.rest).
Step 2: Define the REST Controller Blueprint
  • 1 Right-click on your new rest package, select New -> Java Class.
  • 2 Name the file structure entry layout: SimpleRestController.
  • 3 Populate the empty template container interface by writing the tracking code parameters precisely as shown below:
package com.example.springboot.demo.rest; import org.springframework.web.bind.annotation.GetMapping; import org.springframework.web.bind.annotation.RestController; @RestController public class SimpleRestController { // Expose a "/" endpoint that returns "my application" @GetMapping("/") public String app() { return "my application"; } }
Key Annotations Explained

@RestController: Tells Spring Boot that this class is a web controller capable of handling incoming web requests and converting returned data bodies directly into clean web responses.

@GetMapping("/"): Listens for standard HTTP GET requests hitting the root directory context line of your application host portal.

Step 3: Launch and Verify Endpoint Lifecycle Hooks
  • 1 Re-run the main framework engine by executing your primary class loader launcher file (MyappApplication.java).
  • 2 Open a web browser tab context frame and visit your local testing portal: http://localhost:8080.
  • 3 Verify that the default Whitelabel Error message layout has been replaced with the plaintext output string: my application.
Success! Your Spring Boot ecosystem is now actively listening to requests and dynamically sending response payloads back to clients.
